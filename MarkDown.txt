Абзац:
пустая строка.
Чтобы сделать перенос строки вместо абзаца,
нужно поставить два пробела в конце предыдущей строки.

Заголовки:
# Заголовок первого уровня #

## Заголовок h2

### Заголовок h3

#### Заголовок h4

##### Заголовок h5

###### Заголовок h6

Списки:
Для разметки неупорядоченных списков можно использовать или *, или -, или +
Упорядоченный список:


1. элемент 1

2. элемент 2

    1. вложенный
Вложенные пункты создаются четырьмя пробелами перед маркером пункта:


* элемент 1

* элемент 2

    * вложенный элемент 2.1
Список с абзацами:


* Раз абзац.

* Два абзац.

* Три абзац.

Цитаты:
Цитаты оформляются с помощью символа >

Код:
Исходный код


В чистом Маркдауне блоки кода отбиваются 4 пробелами в начале каждой строки.


Но в GitHub-Flavored Markdown (GFM) есть более удобный способ: ставим по три апострофа до и после кода. 
Также можно указать язык исходного кода.

Инлайн код


Для вставки кода внутри предложений нужно заключать этот код в апострофы.
Если внутри кода есть апостроф, то код надо обрамить двойными апострофами.

Горизонтальная черта:
***
или
---

Ссылки:

Это встроенная [ссылка с title элементом](http://example.com/link "Я ссылка"). 
Это — [без title](http://example.com/link).

Вдыделение:
Выделять слова можно при помощи `*` и `_`. 
Одним символ для наклонного текста, два символа для жирного текста, три — для наклонного и жирного одновременно.

Зачеркивание:

В GFM добавлено зачеркивание текста: две тильды `~` до и после текста.
